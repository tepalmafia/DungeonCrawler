# 게임 기획안 — 《던전 크롤러: 심연의 탑》 (가제)

> **버전**: 1.0 (2026-07-23)
> **제작 원칙**: 외부 프로그램(Unity 에디터, 포토샵, 사운드 툴 등) 일절 없이 **클로드코드 단독으로 코드·그래픽·사운드·데이터를 100% 생성**하고, 브라우저에서 즉시 실행·검증 가능한 게임을 만든다.

---

## 1. 개요

| 항목 | 내용 |
|---|---|
| 장르 | 로그라이크 던전 크롤러 액션 RPG (2D 탑다운) |
| 플랫폼 | 웹 브라우저 (데스크톱 우선, HTML5 Canvas) |
| 플레이 시간 | 1회 런(run) 20~40분, 메타 성장 포함 총 15시간+ |
| 타겟 유저 | 빌드 조합·성장·반복 플레이를 즐기는 RPG/로그라이크 팬 |
| 레퍼런스 | Hades(런 구조·메타 성장), 뱀서라이크(빌드 시너지), 디아블로(루트 도파민), 젤다 초기작(탑다운 전투) |
| 기술 스택 | 순수 HTML5 + Canvas 2D + Web Audio API + Vanilla JavaScript (빌드 도구·외부 라이브러리 없음) |

### 왜 이 조합인가 — "클로드코드 단독 제작" 제약의 해법

| 일반적 게임 제작 요소 | 외부 툴 의존 | 이 기획의 해법 |
|---|---|---|
| 그래픽 (스프라이트) | 포토샵/에이스프라이트 | **코드로 픽셀아트 생성** — 스프라이트를 문자열 배열(픽셀맵)로 정의하고 Canvas에 렌더링. 팔레트 스왑으로 적 변종 무한 생성 |
| 사운드 | 오디오 툴/에셋 스토어 | **Web Audio API로 절차 생성** — 타격음, 레벨업 팡파레, 배경 앰비언트를 오실레이터+엔벨로프로 합성 |
| 맵/레벨 | 에디터 씬 배치 | **절차적 던전 생성** — 시드 기반 알고리즘으로 매번 다른 던전. 레벨 디자인이 데이터가 아니라 코드 |
| 게임 실행/테스트 | 엔진 빌드 | **브라우저에서 즉시 실행** — index.html 하나 열면 끝. 클로드코드가 Playwright로 직접 스크린샷 검증 가능 |
| 콘텐츠 볼륨 | 아티스트/기획 인력 | **데이터 주도 설계** — 아이템·스킬·적을 전부 JS 데이터 테이블로 정의. 코드 몇 줄 추가로 콘텐츠 확장 |

> 기존 저장소의 Unity 프로젝트(Assets/)는 그대로 두고, 웹 게임은 별도 폴더(`web/`)에서 개발한다. 추후 Unity로 이식할 경우 이 기획안과 데이터 테이블을 그대로 재사용할 수 있다.

## 2. RPG 유저는 무엇을 즐기는가 — 기획의 근거

이 기획안의 모든 시스템은 아래 5가지 핵심 재미에서 역산해 설계했다.

1. **성장의 체감 (Progression)** — "아까는 못 잡던 적을 지금은 쉽게 잡는다"는 감각. 숫자가 오르는 것보다 *체감*이 중요하다.
   → 런 내 성장(레벨업·아이템) + 런 간 성장(메타 업그레이드) 이중 구조.
2. **빌드의 자유와 시너지 (Build Crafting)** — 내가 고른 선택지가 조합되어 예상 밖의 강력함을 만드는 순간.
   → 스킬·유물·무기가 서로 태그로 상호작용하는 시너지 시스템.
3. **루트의 도파민 (Loot)** — 상자를 열 때의 기대감, 희귀 등급이 떴을 때의 쾌감.
   → 등급(커먼~레전더리) + 랜덤 옵션 + 시각·청각 연출로 획득 순간을 강화.
4. **의미 있는 선택과 리스크 (Risk & Reward)** — "체력이 없는데 저 엘리트 방에 들어갈까?"
   → 갈림길 구조의 방 선택, 저주받은 상자, 체력을 지불하는 계약 등.
5. **죽어도 남는 것 (Meta Progression)** — 실패한 런도 헛되지 않다는 위안과 재도전 동기.
   → 사망 시에도 유지되는 재화 '영혼 파편'으로 영구 업그레이드·신규 직업 해금.

## 3. 핵심 게임 루프

```
[거점: 탑 입구]
   │  영구 업그레이드 구매, 직업 선택
   ▼
[런 시작: 1층 진입]
   │
   ▼
[방 클리어 루프] ◄─────────────┐
   │ 적 전멸 → 보상 (셋 중 택1)    │
   │ 다음 방 갈림길 선택           │
   │ (전투방/엘리트/상점/이벤트/보물) │
   ▼                          │
[층 보스] ──────────────────────┘ (3~5개 방마다)
   │ 처치 → 유물 획득 → 다음 층
   ▼
[최종 보스: 5층] → 클리어 or 사망
   │
   ▼
[결과 정산] → 영혼 파편 획득 → 거점으로 (반복)
```

- **한 층 = 방 8~12개**, 갈림길에서 다음 방의 종류를 미리 보고 선택 (Slay the Spire식 리스크 관리).
- **한 런 = 5층**, 20~40분. "한 판만 더"를 부르는 길이.

## 4. 전투 시스템 (실시간 액션)

### 4.1 기본 조작
| 입력 | 동작 |
|---|---|
| WASD / 방향키 | 8방향 이동 |
| 마우스 좌클릭 / J | 기본 공격 (마우스 방향) |
| 스페이스 / Shift | 대시 (무적 프레임 0.2초, 쿨다운 1.5초) |
| 우클릭 / K | 스킬 1 (직업 고유) |
| Q / L | 스킬 2 (궁극기, 게이지 충전식) |

### 4.2 전투 감각 (Game Feel) — 최우선 품질 목표
액션 RPG의 품질은 타격감이 결정한다. 코드만으로 구현 가능한 요소에 집중:
- **히트스톱**: 타격 시 40~80ms 게임 일시정지
- **화면 흔들림**: 피격·강공격 시 카메라 셰이크
- **넉백**: 공격 방향으로 적 밀림
- **피격 플래시**: 적 스프라이트 흰색 점멸
- **데미지 숫자**: 크리티컬은 크고 노랗게 튀어오름
- **파티클**: 타격 스파크, 사망 시 픽셀 파편 (모두 코드 생성)
- **타격음**: Web Audio 합성음, 크리티컬은 피치 상승

### 4.3 스탯
| 스탯 | 효과 |
|---|---|
| 체력(HP) | 0이 되면 런 종료 |
| 공격력(ATK) | 기본 데미지 |
| 공격속도(SPD) | 공격 쿨다운 감소 |
| 크리티컬(CRIT) | 확률·배율 (기본 5% / 150%) |
| 이동속도(MOV) | 이동·대시 거리 |
| 행운(LUK) | 드랍 등급 확률, 이벤트 결과 보정 |

## 5. 직업 (플레이어블 캐릭터)

출시 시 3종, 메타 해금으로 확장. 각 직업은 조작 감각 자체가 달라야 한다.

| 직업 | 해금 | 기본 공격 | 스킬 1 | 궁극기 | 컨셉 |
|---|---|---|---|---|---|
| **검사** | 기본 | 3연격 근접 베기 (3타째 광역) | 회전 베기 | 검기 폭풍 (주변 연속타) | 근접 표준형, 입문용 |
| **궁수** | 파편 300 | 차지 화살 (누를수록 관통) | 후방 점프샷 | 화살비 (지정 범위 폭격) | 원거리 카이팅 |
| **마도사** | 파편 800 | 유도 마탄 | 화염 지대 설치 | 시간 정지 3초 | 설치·컨트롤형 |

- 직업별 시작 유물 1개 고정 → 빌드의 출발점 차별화.
- 확장 후보: 도적(백스탭·은신), 성기사(방패·반격), 소환사(하수인).

## 6. 성장 시스템 (이중 구조)

### 6.1 런 내 성장 — 레벨업 & 스킬 선택
- 적 처치 → 경험치 → 레벨업 시 **랜덤 3개 중 1개 선택** (뱀서라이크 방식).
- 선택지 풀: 스탯 강화(소), 액티브 스킬 강화, **패시브 특성**(핵심).
- 패시브 특성은 **태그 시너지**로 설계:

| 태그 예시 | 특성 예시 | 시너지 예시 |
|---|---|---|
| 🔥 화염 | 공격 시 20% 확률 점화 | 점화 + "화상 중인 적에게 크리 확정" + "적 사망 시 폭발" = 화염 연쇄 빌드 |
| ⚡ 번개 | 대시가 경로에 감전 남김 | 감전 + "감전된 적 처치 시 쿨다운 초기화" = 대시 무한 빌드 |
| 🩸 흡혈 | 크리티컬 시 HP 1 회복 | 흡혈 + "HP 만땅일 때 공격력 +30%" = 유지력 딜러 빌드 |
| 🛡️ 수호 | 피격 시 3초 보호막 | 보호막 + "보호막 있는 동안 반사 데미지" = 탱커 빌드 |

- **설계 규칙**: 모든 특성은 최소 2개 이상의 다른 특성과 시너지가 나야 채용된다. 단독으로만 좋은 특성 금지.

### 6.2 런 간 성장 — 메타 프로그레션
- 사망/클리어 시 **영혼 파편** 획득 (도달 층수·처치 수 비례).
- 거점의 "기억의 제단"에서 영구 업그레이드:

| 트리 | 내용 | 목적 |
|---|---|---|
| 육체 | 시작 HP/ATK 소폭 증가 (10단계) | 실력 부족을 시간으로 보완하는 완만한 파워커브 |
| 기회 | 레벨업 선택지 3→4개, 상점 리롤 +1 | 빌드 자유도 확장 |
| 유산 | 신규 직업·유물 풀 해금 | 콘텐츠 개방 |
| 도전 | 고난이도 모드(열기 단계) 해금 | 클리어 이후의 엔드 콘텐츠 |

- **밸런스 원칙**: 메타 성장 총합은 초반 체감 +30% 이내. "메타를 다 찍어야 깨는 게임"이 아니라 "실력이 늘어서 깨는 게임"이 되도록.

## 7. 아이템 & 유물

### 7.1 등급
| 등급 | 색 | 드랍률 | 연출 |
|---|---|---|---|
| 커먼 | 흰색 | 60% | 기본 |
| 레어 | 파랑 | 27% | 빛 기둥 |
| 에픽 | 보라 | 10% | 빛 기둥 + 전용 사운드 |
| 레전더리 | 주황 | 3% | 화면 이펙트 + 팡파레, 규칙을 비트는 효과 |

### 7.2 유물 (패시브 장비, 런 한정)
- 층 보스·보물방·상점에서 획득, 슬롯 무제한.
- 레전더리는 수치 강화가 아니라 **게임 규칙 변경**: "체력이 1이 되는 대신 모든 공격 크리티컬", "대시 무한, 대신 이동속도 -30%" 등 빌드를 뒤집는 효과.
- **저주 유물**: 강력한 효과 + 디메리트. 저주받은 상자에서만 등장 (리스크&리워드).

### 7.3 상점
- 층당 1회 등장. 유물·회복·리롤 판매. 재화 '금화'는 런 종료 시 소멸 (파편과 분리) → 런 내에서 아낌없이 쓰게 유도.

## 8. 던전 절차 생성

### 8.1 구조
- **층 맵**: 방 노드 그래프 생성 → 갈림길 2~3개 분기 → 보스방으로 수렴. 플레이어는 다음 방 아이콘(전투/엘리트/상점/이벤트/보물)을 보고 선택.
- **방 내부**: 시드 기반 타일맵 생성 (벽·장애물·함정 배치). 방 크기 3종(소/중/대) × 장애물 패턴 풀 조합.
- **시드 공개**: 결과 화면에 시드 표시, 시드 입력 플레이 지원 (같은 던전 재도전·공유 가능).

### 8.2 층 테마 (팔레트 스왑 + 규칙 변화로 코드 재사용)
| 층 | 테마 | 고유 규칙 | 등장 적 |
|---|---|---|---|
| 1층 | 지하 묘지 | 없음 (튜토리얼 겸용) | 슬라임, 해골 |
| 2층 | 곰팡이 동굴 | 독 안개 지대 | 버섯, 박쥐, 독거미 |
| 3층 | 잊힌 감옥 | 감옥 문 개폐 기믹 | 간수 골렘, 망령 |
| 4층 | 용암 심층 | 바닥 용암 (지형 데미지) | 화염 정령, 용암 개 |
| 5층 | 심연의 옥좌 | 어둠 (시야 제한) | 정예 혼합 + 최종 보스 |

## 9. 적 & 보스 설계

### 9.1 적 설계 원칙
- 모든 적은 **공격 전 예고 동작(텔레그래프)** 필수 — 색 점멸, 조준선, 바닥 범위 표시. "이유 있는 죽음"만 허용.
- 역할 조합으로 전투 다양성: 돌진형 + 원거리형 + 소환형 + 방패형을 섞어 배치.

| 유형 | 행동 | 대응법 |
|---|---|---|
| 슬라임 (기본) | 느린 추적, 접촉 데미지 | 기본 처치 연습 |
| 해골 궁수 | 거리 유지 + 조준선 표시 후 발사 | 대시로 회피, 우선 처치 |
| 돌진 멧돼지 | 1초 조준 후 직선 돌진, 벽에 박으면 그로기 | 유인 후 반격 |
| 네크로맨서 | 잡몹 소환, 도주 | 본체 집중 |
| 방패 골렘 | 정면 무적 | 대시로 배후 잡기 |

### 9.2 보스 (층당 1)
- **다단계 페이즈**: HP 50% 이하에서 패턴 추가·강화.
- **패턴은 3~5개 조합의 명확한 리듬**: 외우면 노히트가 가능하되, 처음 보면 반드시 긴장하는 난이도.
- 예시 — 1층 보스 "무덤지기 카론": ①낫 3연 휘두르기(근접) ②영혼 투사체 부채꼴 ③(50% 이하) 바닥 저주 지대 + 잡몹 소환.

## 10. UI / UX

- **HUD**: 좌상단 HP바·경험치바, 우상단 층수·금화, 하단 스킬 쿨다운 아이콘, 획득 특성은 좌측에 태그 아이콘 스택.
- **레벨업 화면**: 게임 일시정지 + 카드 3장 제시. 카드에 태그 아이콘 표시 → 시너지를 한눈에.
- **사망 화면**: 이번 런 요약(도달 층, 처치 수, 획득 빌드), 파편 정산 애니메이션 → 즉시 "다시 도전" 버튼. **재도전까지 3클릭 이내.**
- **첫 플레이**: 별도 튜토리얼 없이 1층 초반 방에서 이동→공격→대시를 자연스럽게 가르치는 배치.
- 키보드 온리 / 키보드+마우스 모두 지원.

## 11. 아트 & 사운드 방향 (100% 코드 생성)

### 11.1 아트
- **16×16 픽셀아트**, 3배 확대 렌더링(픽셀 퍼펙트, `imageSmoothingEnabled = false`).
- 스프라이트는 문자열 픽셀맵으로 코드에 정의:
  ```js
  // 예: 슬라임 (문자 = 팔레트 인덱스)
  const SLIME = [
    "..GGGG..",
    ".GGGGGG.",
    "GGWGGWGG",   // W = 눈 하이라이트
    "GGGGGGGG",
  ];
  ```
- **팔레트 스왑**으로 적 변종(일반/엘리트/층 테마별)을 데이터만으로 생성.
- 애니메이션은 2~4프레임 교체 + 코드 변형(스쿼시&스트레치, 기울임)으로 생동감.
- 층별 통일 팔레트(어두운 배경 + 채도 높은 캐릭터)로 가독성 확보.

### 11.2 사운드
- **Web Audio API 합성**: 타격(노이즈+짧은 감쇠), 크리티컬(피치업), 레벨업(상승 아르페지오), 피격(저음 펄스), 보스 등장(불협화음 스팅어).
- BGM은 층 테마별 코드 진행을 가진 절차 생성 루프 (베이스+아르페지오+드럼, 미니멀 던전 신스).
- 음소거·볼륨 설정 제공.

## 12. 기술 아키텍처

```
web/
├── index.html            # 진입점 (이것만 열면 실행)
├── css/style.css
└── js/
    ├── main.js           # 게임 루프 (requestAnimationFrame, 고정 타임스텝)
    ├── core/             # 엔진 계층
    │   ├── renderer.js   #   Canvas 렌더링, 카메라, 셰이크
    │   ├── input.js      #   키보드/마우스
    │   ├── audio.js      #   Web Audio 합성기
    │   ├── particles.js  #   파티클 시스템
    │   └── rng.js        #   시드 기반 난수 (mulberry32)
    ├── game/             # 게임 로직 계층
    │   ├── player.js  enemy.js  boss.js  combat.js
    │   ├── dungeon.js    #   절차 생성
    │   ├── progression.js#   레벨업/특성/시너지
    │   └── meta.js       #   영혼 파편, localStorage 저장
    ├── data/             # 콘텐츠 계층 (밸런싱은 여기만 수정)
    │   ├── sprites.js  enemies.js  items.js  traits.js  floors.js
    └── ui/
        ├── hud.js  menus.js  levelup.js
```

- **외부 의존성 0** — CDN·npm 없이 파일만으로 동작 (오프라인 실행 가능).
- **세이브**: localStorage (메타 진행도 + 설정).
- **검증 루프**: 클로드코드가 Playwright로 게임을 직접 구동·스크린샷·전투 시뮬레이션하여 밸런스와 렌더링을 자체 검증.
- 성능 목표: 60fps, 화면 내 적 30 + 파티클 300 동시 처리.

## 13. 개발 로드맵

| 마일스톤 | 내용 | 완료 기준 |
|---|---|---|
| **M1. 코어 전투** | 이동·공격·대시, 적 3종, 타격감(히트스톱·셰이크·파티클), 픽셀 렌더러, 기본 사운드 | "전투만으로 재밌다"는 감각. 방 하나에서 10분 놀 수 있으면 통과 |
| **M2. 던전 루프** | 절차 생성 1층, 방 선택 갈림길, 레벨업·특성 12종, 1층 보스 | 1층 클리어까지 온전한 런 1회 가능 |
| **M3. 성장의 깊이** | 5층 전체, 적 12종+보스 5종, 특성 30종·유물 20종, 시너지 시스템 | 서로 다른 빌드 3개로 클리어 경험이 달라짐 |
| **M4. 메타 & 거점** | 영혼 파편, 영구 업그레이드, 직업 3종, 사망/정산 화면, 세이브 | 죽어도 다시 하고 싶은 루프 완성 |
| **M5. 폴리싱** | BGM, 층 테마 아트, 밸런스 조정, 열기(고난이도) 모드, 시드 공유 | 외부인에게 보여줄 수 있는 완성도 |

각 마일스톤은 독립적으로 플레이 가능한 상태로 완결한다 (항상 실행되는 게임 유지).

## 14. 성공 기준
