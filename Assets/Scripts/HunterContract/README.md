# 헌터의 의뢰 — 프로토타입

기획서: `Docs/GameDesign.md` (§8 첫 프로토타입 범위 기준 구현)

## 실행 방법

1. Unity에서 **새 씬**을 만든다 (File → New Scene). 기존 SampleScene의 Player/Enemy와 섞이지 않게 새 씬 권장.
2. 빈 GameObject를 만들고 **`GameBootstrap`** 컴포넌트를 추가한다.
3. **Play**. 카메라·아레나·UI·게임 루프가 전부 코드로 생성된다.

씬 에셋·프리팹·스프라이트 없이 동작한다 (스프라이트·폰트는 런타임 생성, 한글은 OS 폰트 자동 탐색).

## 구현 범위 (기획서 §8)

- **루프 전체 1회전**: 의뢰 게시판 → 준비(상점+감빗) → 함정 배치 → 사냥(3분) → 정산 → 반복
- **몬스터 2종**: 멧돼지 왕(튜토리얼: 조준→돌진→경직), 질풍 사슴(도주형: 거리 유지·질주·탈출구 도주)
- **함정 3종**: 속박 덫(4초 속박, 도주 저지), 전격 트랩(2초 기절+피해), 가시 지대(지속 도트+슬로우)
- **감빗**: 기본 조건 4종 × 행동 4종, 슬롯 3개, 우선순위 위→아래
- **긴급 카드 3종**: 함정 재가동 / 집중 사격 / 긴급 미끼(클릭 지점으로 강제 유인)
- **핵심 규칙**: 함정 걸림 피해 2배, 헌터 기절 10초 후 부활(몬스터는 경계로 복귀), 도주 저지 시 게이지 리셋, 보상 배율(시간/카드/노기절)

## 프로토타입에서 생략된 것 (본편 기획에는 있음)

- 정찰/도감 시스템 (의뢰 정보는 전부 공개 상태)
- 부위 파괴 보너스
- 메타 저장 (명예 pt는 에디터 세션 동안만 누적)
- 몬스터 3~5번, 랭크 승급, 감빗 확장 슬롯

## 파일 구성

| 파일 | 역할 |
|---|---|
| `GameBootstrap.cs` | 진입점 — 카메라·아레나·매니저·UI 생성 |
| `GameFlow.cs` | 페이즈 흐름(의뢰→준비→배치→사냥→정산), 상점, 함정 배치 입력 |
| `HuntManager.cs` | 3분 타이머, 승패 판정, 긴급 카드, 보상 정산 |
| `MonsterAI.cs` | 공통 상태머신(경계/교전/도주/함정 걸림) + 멧돼지·사슴 패턴 |
| `HunterAI.cs` | 감빗 평가·실행, 자동 이동/공격, 기절·부활 |
| `Gambit.cs` | 조건·행동 정의 |
| `Trap.cs` | 함정 3종 + 미끼 |
| `GameDefs.cs` | 몬스터/가격/카드 데이터, 판 상태(RunState) |
| `CombatUnit.cs` | HP 홀더 (기존 Health.cs와 별개) |
| `GameUI.cs` | 전 UI 코드 생성 (uGUI) |
| `SpriteFactory.cs` | 런타임 스프라이트·폰트·이펙트 |

기존 스크립트(`PlayerMovement` 등)는 수정하지 않았다.

## 밸런스 튜닝 포인트

- 몬스터 스탯·도주 조건: `GameDefs.cs`의 `DB.Boar` / `DB.Deer`
- 헌터 스탯·행동 수치: `HunterAI.cs` 상단 상수 및 `RunAction`
- 함정 수치: `Trap.cs`, 가격: `GameDefs.DB.TrapPrice`
- 타이머·보상 배율: `HuntManager.cs`
